总结建议​

  • 优先选择：Docker 容器化部署（节省资源）
  • 内存敏感：限制单服务内存（如docker run --memory 2g）
  • 性能瓶颈：避免同时运行 3 个以上高内存服务（如 LLM+DB+媒体转码）

通过合理分配，16GB 内存可同时承载约 10-15 个中低负载服务。
